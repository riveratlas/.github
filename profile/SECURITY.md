## Reporting a Vulnerability

### Private Disclosure Process

We take all security vulnerabilities seriously. Thank you for improving the security of our open-source software. We appreciate your efforts and responsible disclosure and will make every effort to acknowledge your contributions.

To report a security vulnerability, please email us at [security@riveratlas.org](mailto:security@riveratlas.org) with the subject line "[SECURITY] Vulnerability Report".

**Please do not report security vulnerabilities through public GitHub issues, discussions, or pull requests.**

### Required Information

Please include the following information in your report:

1. Project name and version
2. Description of the vulnerability
3. Steps to reproduce the issue
4. Impact of the vulnerability
5. Any known mitigations
6. Your name and affiliation (if any)

### Response Time

We will acknowledge receipt of your vulnerability report within 48 hours and will send a more detailed response within 72 hours indicating the next steps in handling your report.

### Public Disclosure

After the vulnerability has been fixed, we will publish a security advisory on GitHub. We will credit you for your discovery, unless you prefer to remain anonymous.

## Security Best Practices

### For Users
- Always keep your dependencies up to date
- Review the security documentation for each project
- Follow the principle of least privilege
- Use strong, unique passwords
- Enable two-factor authentication on your accounts

### For Contributors
- Follow secure coding practices
- Keep dependencies up to date
- Review all third-party code before including it in the project
- Write and maintain tests, including security tests
- Follow the principle of least privilege in all code
